# 计费逻辑说明

## 概述

本系统使用"灵感点"（字数点）作为计费单位，用户调用 AI 生成内容时会消耗灵感点。

## 计费公式

```
消费灵感点 = (输入token/10 + 输出token) × 模型倍率
```

### 参数说明

| 参数 | 说明 |
|------|------|
| 输入token | 按字符数计算，中文1字符≈1token，英文约4字符≈1token |
| 输出token | AI生成内容的字符数 |
| 模型倍率 | 不同模型有不同的成本倍率，默认为1.0 |

### 为什么输入token除以10？

- AI写作场景中，输入通常包含大量上下文（小说设定、前文、章纲等）
- 输入内容往往是输出的数倍甚至数十倍
- 为了降低用户成本，输入token除以10后再计算

## 计费示例

### 示例1：章节生成
- 输入：10000 字符（包含上下文、设定等）
- 输出：3000 字符（生成的章节内容）
- 模型倍率：1.0

计算：
```
(10000/10 + 3000) × 1.0 = (1000 + 3000) × 1.0 = 4000 点
```

### 示例2：使用高级模型
- 输入：5000 字符
- 输出：2000 字符
- 模型倍率：1.5（高级模型）

计算：
```
(5000/10 + 2000) × 1.5 = (500 + 2000) × 1.5 = 3750 点
```

## 扣费流程

1. **预检查**：调用AI前，检查用户余额是否足够支付预估输出费用（约4000点）
   - 预检查只检查最低余额要求，不包含输入成本
   - 这样避免因为大量上下文导致预估过高
2. **生成内容**：调用AI生成内容
3. **实际扣费**：根据实际输入/输出字符数计算并扣费
4. **记录交易**：记录交易详情（输入token、输出token、模型、消费金额等）

## 相关代码

- 计费计算：`SystemAIConfigService.calculateCost()`
- 扣费服务：`CreditService.consume()`
- AI写作扣费：`AIWritingService` 中的各个生成方法

## 模型倍率配置

模型倍率在 `ai_models` 表的 `cost_multiplier` 字段配置，管理员可在后台调整。

| 模型类型 | 建议倍率 |
|---------|---------|
| 基础模型 | 1.0 |
| 高级模型 | 1.5 - 2.0 |
| 顶级模型 | 2.0 - 3.0 |

## 注意事项

1. 余额不足时会抛出 `InsufficientCreditsException` 异常
2. 预检查只检查最低余额（预估输出费用），实际扣费按真实消耗计算
3. 扣费失败不会阻断内容生成，但会记录警告日志
4. 所有交易都会记录到 `credit_transactions` 表
