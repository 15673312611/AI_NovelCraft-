# 404 错误修复说明

## 问题描述
前端详情页所有接口都返回 404 错误。

## 问题原因

### 1. 路径配置问题
- **前端 baseURL**: `/admin`
- **前端请求**: `/api/admin/novels/{id}`
- **实际请求**: `/admin/api/admin/novels/{id}` ❌ (错误)

### 2. Controller 路径冲突
- `AdminNovelController`: `@RequestMapping("/novels")`
- `NovelDetailController`: `@RequestMapping("/api/admin/novels")`
- 两个 Controller 都有 `/{id}` 路径，导致冲突

### 3. 路径映射不一致
前端期望的路径和后端实际提供的路径不匹配。

## 解决方案

### 1. 合并 Controller
将 `NovelDetailController` 的所有接口合并到 `AdminNovelController` 中：

```java
@RestController
@RequestMapping("/novels")
@CrossOrigin(origins = "*")
@RequiredArgsConstructor
public class AdminNovelController {
    private final AdminNovelService novelService;
    private final GraphDataService graphDataService;
    private final NovelDetailService novelDetailService; // 新增
    
    // 原有接口
    @GetMapping("/{id}")
    public NovelDTO getNovelById(@PathVariable Long id) { ... }
    
    // 新增接口
    @GetMapping("/{novelId}/outline")
    public Map<String, Object> getOutline(@PathVariable Long novelId) { ... }
    
    @GetMapping("/{novelId}/volumes")
    public Map<String, Object> getVolumes(@PathVariable Long novelId) { ... }
    
    @GetMapping("/{novelId}/chapter-outlines")
    public Map<String, Object> getChapterOutlines(...) { ... }
    
    @GetMapping("/{novelId}/chapters")
    public Map<String, Object> getChapters(...) { ... }
    
    @GetMapping("/{novelId}/detail-all")
    public Map<String, Object> getDetailAll(@PathVariable Long novelId) { ... }
    
    @GetMapping("/{novelId}/statistics")
    public Map<String, Object> getStatistics(@PathVariable Long novelId) { ... }
}
```

### 2. 修改前端请求路径
将前端请求从 `/api/admin/novels` 改为 `/novels`：

```typescript
// 修改前
axios.get(`/api/admin/novels/${novelId}`)

// 修改后
axios.get(`/novels/${novelId}`)
```

### 3. 删除独立的 Controller
删除 `NovelDetailController.java`，避免路径冲突。

## 注意事项

1. **路径一致性**
   - 前端 baseURL + 请求路径 = 后端完整路径
   - 确保所有路径配置一致

2. **Controller 合并**
   - 避免多个 Controller 使用相同的路径前缀
   - 使用子路径区分不同功能

3. **依赖注入**
   - 使用 `@RequiredArgsConstructor` 时，所有 `final` 字段都会被注入
   - 确保 Service 类已正确配置

4. **响应格式**
   - 统一使用 `{ code: 200, data: {...} }` 格式
   - 前端需要兼容处理 `response.data?.data || response.data`
