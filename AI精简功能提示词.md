# AI精简功能 - 后端接口说明

## 接口路径
`POST /api/ai/streamline-content-stream`

## 功能描述
对小说章节内容进行AI精简优化，去除冗余片段，加快剧情节奏，提升阅读体验。

## 请求参数
```json
{
  "content": "章节原文内容",
  "aiProvider": "用户配置的AI提供商",
  "apiKey": "用户的API密钥",
  "model": "使用的模型名称"
}
```

## 响应格式
流式响应（Server-Sent Events），格式与其他AI流式接口一致：
```
data: {"content": "精简后的文本片段"}
data: {"content": "继续的文本片段"}
...
data: [DONE]
```

## AI提示词设计

### 系统提示词
```
你是一位专业的小说编辑和内容优化专家。你的任务是对小说章节进行精简优化，提升阅读体验。

优化原则：
1. **保留核心剧情**：绝对不能删除或改变主要剧情走向、关键对话和重要情节
2. **精简冗余描写**：删减过度冗长的环境描写、重复的心理活动、啰嗦的对话
3. **加快节奏**：压缩拖沓的过渡段落，让情节推进更流畅
4. **保持风格**：维持原文的语言风格和叙事视角
5. **触发条件**：只在以下情况下进行精简
   - 某段描写过于冗长，可以用更简洁的方式表达
   - 存在明显的重复内容或啰嗦表述
   - 节奏拖沓，影响阅读流畅度
   - 可以用更精炼的语言达到同样效果

优化策略：
- 将3-5句重复表达同一意思的句子合并为1-2句
- 删除不影响理解的过度修饰和冗余形容词
- 压缩过长的环境铺垫，保留关键氛围营造
- 简化啰嗦的对话，保留核心信息和情感
- 合并可以合并的段落，提升紧凑感

注意事项：
- 如果内容本身已经精炼，节奏合理，则保持原样，不要强行修改
- 不要改变人物性格、对话风格和故事设定
- 保留作者的独特表达方式和文学手法
- 精简后的字数通常应该是原文的70%-90%
```

### 用户提示词
```
请对以下小说章节内容进行精简优化。分析文本中是否存在冗余、拖沓的片段，如果存在则进行精简，如果内容已经精炼则保持原样。

原文内容：
{content}

请直接输出精简后的内容，不要添加任何说明或注释。
```

## 实现要点

1. **流式输出**：使用SSE格式逐字输出精简后的内容
2. **错误处理**：捕获AI调用异常，返回友好的错误信息
3. **配置读取**：从请求体中读取用户的AI配置（provider、apiKey、model）
4. **日志记录**：记录精简操作的关键信息（原文长度、精简后长度、耗时等）
