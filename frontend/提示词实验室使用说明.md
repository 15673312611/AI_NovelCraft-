# 提示词实验室使用说明

## 功能概述

提示词实验室是一个多AI协作优化提示词的工具。通过让多个AI模型扮演不同角色（分析师、批评家、创造者、优化师），对您的提示词进行多轮讨论和优化。

## 主要功能

### 1. 提示词输入
- **原始提示词**：输入您想要优化的提示词
- **优化目标**：可选，说明您希望解决的问题（例如："现在的提示词生成的内容缺乏逻辑性"）

### 2. AI团队配置
系统默认配置了三个AI角色，都使用 `deepseek-chat` 模型：
- 🔍 **分析师**：分析提示词的结构、意图和潜在问题，列出优缺点
- ⚠️ **批评家**：批评提示词和其他AI的分析，找出逻辑漏洞和问题，并可以反驳其他AI的错误建议
- 💡 **创造者**：头脑风暴，提供创意角度和补充元素
- ✨ **优化师**：综合所有讨论，生成优化后的提示词

您可以：
- 添加更多AI角色
- 编辑现有角色的配置（模型名称、角色类型）
- 为不同角色配置不同的模型
- 启用/禁用特定角色
- 删除不需要的角色

> **重要**：所有AI角色共用全局API地址和密钥（在"设置"页面配置），但每个角色可以使用不同的模型名称。

### 3. 讨论流程

#### 工作原理
1. **第一轮讨论**：
   - AI1（分析师）首先发言，分析提示词的优点和问题
   - AI2（批评家）查看AI1的分析，指出问题或反驳错误观点
   - AI3（优化师）综合前面的讨论，提出优化建议
   - 其他AI模型依次发言...

2. **第二轮讨论**：
   - AI1看到后面AI的反馈，可以反驳或同意
   - AI2继续批评和指正
   - 每个AI都能看到之前所有的讨论内容
   - 形成真正的"讨论"而不是单向输出

3. **最终优化**：
   - 优化师AI会根据所有讨论内容，生成最终的优化提示词
   - 优化后的提示词会显示在特殊的卡片中

### 4. 用户反馈与继续优化

如果您对优化结果不满意：
1. 在反馈框中说明问题（例如："优化后的提示词太长了"）
2. 点击"进入下一轮优化"
3. AI团队会基于您的反馈继续讨论和优化
4. 可以进行多轮优化直到满意

### 5. 历史记录

所有实验会话都保存在浏览器本地存储中：
- 点击"历史记录"按钮查看过往会话
- 可以加载历史会话继续查看或继续优化
- 可以删除不需要的历史记录
- 最多保存50条历史记录

## 使用步骤

1. **配置API**：在"设置"页面配置API地址、密钥和模型（必须）
2. **输入提示词**：在左侧输入您要优化的提示词
3. **设置目标**（可选）：说明您希望解决的具体问题
4. **配置AI团队**：确保至少启用一个AI角色
5. **设置轮数**：选择讨论轮数（1-5轮）
6. **开始实验**：点击"开始实验"按钮
7. **观察讨论**：实时查看AI们的讨论过程
8. **查看结果**：查看优化后的提示词
9. **继续优化**（可选）：如果不满意，提供反馈并继续优化

## 配置AI角色

### 全局API配置（必须）
在使用提示词实验室之前，请先在"设置"页面配置：
- **API地址**：例如 `https://api.deepseek.com/v1`
- **API密钥**：您的API密钥

所有AI角色将共用这个API地址和密钥。

### 添加新角色
1. 点击"添加"按钮
2. 填写角色信息：
   - **模型名称**：例如 `deepseek-chat`、`gpt-4`、`claude-3-5-sonnet`（此名称将作为API调用的model参数）
   - **角色类型**：选择该角色的类型（分析师/批评家/创造者/优化师）

### 编辑角色
点击角色旁边的设置图标即可编辑

### 多模型配置
您可以为不同角色配置不同的模型，例如：
- 分析师：`gpt-4` - 擅长深度分析
- 批评家：`claude-3-5-sonnet` - 擅长找问题
- 优化师：`deepseek-chat` - 性价比高

### 角色说明
- **分析师**：客观分析，列出优缺点
- **批评家**：找问题，反驳错误观点
- **创造者**：提供创意和新思路
- **优化师**：综合优化，生成最终版本

## 注意事项

1. **API配置**：使用前必须在"设置"页面配置API，所有角色共用同一API配置
2. **API密钥安全**：API密钥存储在浏览器本地，请注意安全
3. **网络连接**：需要网络连接才能调用AI API
4. **API费用**：使用AI API可能产生费用，请注意控制使用
5. **讨论轮数**：建议2-3轮即可，过多轮次可能导致讨论发散
6. **角色选择**：建议至少包含一个"优化师"角色来生成最终结果

## 示例场景

### 场景1：优化写作提示词
- **原始提示词**："写一篇关于AI的文章"
- **优化目标**："内容太泛，缺乏具体方向"
- **结果**：AI团队会讨论并生成更具体、更有针对性的提示词

### 场景2：修复逻辑问题
- **原始提示词**："你是一个助手，帮我写代码，但不要写太复杂的"
- **优化目标**："逻辑矛盾，不够清晰"
- **结果**：AI团队会指出矛盾之处并提供清晰的版本

## 常见问题

**Q: 为什么AI没有反应？**
A: 请检查"设置"页面的API配置是否正确，包括API地址、密钥和模型名称，以及网络连接是否正常。

**Q: 可以使用哪些AI模型？**
A: 支持所有兼容OpenAI API格式的模型，包括DeepSeek、GPT-4、Claude等。在"设置"页面配置相应的API地址和模型名称即可。

**Q: 可以为不同角色配置不同的API吗？**
A: 目前所有角色共用全局API配置。如需使用不同模型，请在"设置"中切换配置。

**Q: 历史记录保存在哪里？**
A: 保存在浏览器的localStorage中，清除浏览器数据会丢失历史记录。

**Q: 可以导出优化结果吗？**
A: 可以直接复制优化后的提示词，或点击"应用此提示词"按钮。
