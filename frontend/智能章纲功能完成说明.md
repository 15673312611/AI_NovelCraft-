# 智能章纲生成功能 - 完成说明

## ✅ 功能已完整实现

### 核心创新
**问题**：AI一次性生成30章会陷入"平庸化"陷阱（基于扑街小说数据训练）

**解决方案**：
1. **分段生成**：每次生成3-7章，避免长期平庸累积
2. **看点驱动**：先提取核心看点（冲突/反转/高潮），再围绕看点编排
3. **质量筛选**：每批生成后AI自我评分，低分自动重生成
4. **上下文滚动**：只参考前2-3章，避免陷入套路循环

## 🚀 使用方法

### 1. 访问页面
- 左侧导航 → "AI助手" → "智能章纲"
- 或直接访问：`/smart-outline`

### 2. 操作流程
```
1. 选择小说（下拉框）
2. 选择卷（卡片列表）
3. 选择节奏模板（4种可选）
4. 点击"开始智能生成"
5. 等待3-5分钟（自动分段生成+质量检查）
6. 查看结果并保存
```

### 3. 节奏模板说明
| 模板 | 适用类型 | 特点 |
|------|---------|------|
| **波浪式** | 长篇连载 | 张弛有度，高潮-缓冲交替 |
| **递进式** | 爽文、逆袭流 | 压力递增，爽点密集 |
| **悬疑式** | 宫斗、权谋 | 层层剥茧，真相揭露 |
| **三幕式** | 传统叙事 | 起承转合，经典结构 |

## 🔧 技术细节

### API接口
```
POST /volumes/{volumeId}/smart-outline/extract-highlights
POST /volumes/{volumeId}/smart-outline/plan-units
POST /volumes/{volumeId}/smart-outline/generate-unit
POST /volumes/{volumeId}/smart-outline/quality-check
POST /volumes/{volumeId}/smart-outline/optimize
POST /volumes/{volumeId}/smart-outline/save
```

### 生成流程
```
Phase 1: 提取看点（5-15%）
  ↓ 从大纲和卷蓝图提取核心看点
  
Phase 2: 规划单元（15-25%）
  ↓ 根据节奏模板划分情节单元
  
Phase 3: 分批生成（25-75%）
  ↓ 逐单元生成+质量检查+自动优化
  
Phase 4: 完成（100%）
  ↓ 返回高质量章纲列表
```

## ⚠️ 注意事项

### 前提条件
1. ✅ 小说必须有已确认的全书大纲
2. ✅ 卷需要有内容大纲（contentOutline字段）
3. ✅ AI配置正确（API Key、模型等）

### 数据库字段
- `NovelVolume.contentOutline` - 卷的内容大纲（不是blueprint）
- `NovelVolume.chapterStart/chapterEnd` - 计算章节数

### 常见问题
**Q: 小说列表为空？**
A: 检查后端API是否正常，查看浏览器控制台错误

**Q: 生成时间过长？**
A: 正常3-5分钟，超过10分钟检查AI服务响应

**Q: 质量不满意？**
A: 尝试更换节奏模板或完善卷的内容大纲

## 📊 质量保证

### 检测维度
1. **平庸度检测** - 是否存在无冲突的过渡章
2. **钩子强度** - 每章结尾是否有吸引力
3. **冲突密度** - 是否每章都有明确矛盾
4. **可预测性** - 剧情是否过于套路化
5. **信息堆砌** - 是否存在纯背景介绍

### 自动优化
- 质量评分 < 60分 → 自动重生成
- 最多重试2次
- 强制注入冲突/悬念/反转
