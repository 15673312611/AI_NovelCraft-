# 智能章纲生成系统使用说明

## 功能概述

智能章纲生成系统通过**分段生成 + 看点驱动 + 质量筛选**的创新工作流，解决了AI一次性生成长篇章纲时的平庸化问题。

### 核心优势

1. **规避AI平庸化**：不是一次生成30章，而是分批生成（每批3-7章），每次重新注入高质量约束
2. **看点驱动**：先从大纲提取核心看点（冲突、反转、高潮等），再围绕看点编排章节
3. **质量自检**：每批生成后自动评分，低分章纲自动重新生成
4. **上下文滚动**：每次生成只参考前2-3章，避免陷入长期平庸循环

## 使用流程

### 1. 进入页面

- 点击左侧导航栏 **"AI 助手"** → **"智能章纲"**
- 或直接访问：`/smart-outline`

### 2. 选择小说和卷

**步骤：**
1. 在"选择小说"下拉框中选择目标小说
2. 在"选择卷"列表中点击要生成章纲的卷
3. 选择节奏模板（推荐根据小说类型选择）

**节奏模板说明：**

| 模板 | 适用类型 | 特点 |
|------|---------|------|
| **波浪式节奏** | 长篇连载 | 张弛有度，高潮-缓冲-高潮交替 |
| **递进式节奏** | 爽文、逆袭流 | 压力持续递增，爽点密集 |
| **悬疑式节奏** | 宫斗、权谋 | 层层剥茧，真相逐步揭露 |
| **三幕式经典** | 传统叙事 | 起承转合，经典结构 |

### 3. 开始生成

点击 **"开始智能生成"** 按钮，系统将自动执行以下步骤：

#### Phase 1: 提取看点（5-15%）
- 从全书大纲和卷蓝图中提取核心看点
- 识别冲突点、反转点、高潮点、悬念点等
- 预计耗时：10-20秒

#### Phase 2: 规划情节单元（15-25%）
- 根据节奏模板将章节划分为多个情节单元
- 每个单元5-7章，分配不同的看点和强度
- 预计耗时：5-10秒

#### Phase 3: 分批生成章纲（25-75%）
- 逐个单元生成章纲
- 每个单元生成后自动进行质量检查
- 低质量章纲自动重新生成（最多重试2次）
- 预计耗时：每单元30-60秒

#### Phase 4: 质量优化（75-100%）
- 对所有章纲进行最终质量检查
- 自动优化平庸章节
- 预计耗时：20-40秒

### 4. 查看结果

生成完成后，页面会显示：
- **章纲列表**：每章的主线剧情、关键情节点、情感基调
- **质量评分**：每章的质量分数（60-100分）
- **钩子结尾**：每章的悬念设置

### 5. 保存章纲

点击 **"保存到卷"** 按钮，将生成的章纲保存到数据库。

## 生成质量保证

### 质量检测维度

系统会自动检测以下问题：

1. **平庸度检测**：是否存在"过渡章"、"日常章"等无冲突内容
2. **钩子强度**：每章结尾是否有足够吸引力
3. **冲突密度**：是否每章都有明确的矛盾或悬念
4. **可预测性**：剧情是否过于套路化
5. **信息堆砌**：是否存在纯粹的背景介绍章节

### 自动优化机制

当章纲质量评分低于60分时，系统会：
1. 标记具体问题（如"第3章缺乏冲突"）
2. 给出改进建议（如"增加意外事件或人物冲突"）
3. 自动重新生成该章纲
4. 最多重试2次，确保质量达标

## 技术原理

### 为什么AI一次性生成长篇会很垃圾？

**核心问题：**
- AI基于Transformer架构，本质是"推演"而非"创作"
- 训练数据主要来自扑街小说，AI会倾向于生成平庸剧情
- 一次性生成30章时，AI会陷入"平均化"陷阱，失去创意

### 我们的解决方案

**分段生成 + 质量筛选：**
```
传统方式：
[大纲] → AI一次性生成30章 → [平庸章纲]

智能方式：
[大纲] → 提取看点 → 规划单元 
       ↓
   单元1(5章) → 质量检查 → 不达标？重新生成
       ↓
   单元2(6章) → 质量检查 → 不达标？重新生成
       ↓
   单元3(4章) → ...
       ↓
   [高质量章纲]
```

**关键优化：**
1. **上下文窗口限制**：每次生成只看前2-3章，避免长期平庸累积
2. **看点强制注入**：每个单元必须包含指定的冲突/反转/高潮
3. **质量评分机制**：AI自我评估，低分自动重生成
4. **节奏模板约束**：不同单元有不同的强度要求

## 常见问题

### Q1: 生成时间太长怎么办？
**A:** 正常情况下生成30章需要3-5分钟。如果超过10分钟，可能是：
- AI服务响应慢（检查网络或更换API）
- 质量阈值设置过高（默认60分已足够）

### Q2: 生成的章纲还是不满意？
**A:** 可以尝试：
1. 更换节奏模板（不同模板风格差异很大）
2. 完善卷蓝图内容（提供更多剧情方向）
3. 手动编辑生成后的章纲

### Q3: 能否生成超过30章？
**A:** 可以，系统会根据卷的章节数自动调整。但建议：
- 单次生成不超过35章（避免AI疲劳）
- 超过35章建议分多次生成

### Q4: 生成失败怎么办？
**A:** 检查以下条件：
1. 小说是否有已确认的全书大纲
2. 卷是否有蓝图内容
3. AI配置是否正确（API Key、模型等）

- 调整不合理的情节点
- 确保看点分布均匀
