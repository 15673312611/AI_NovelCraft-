# 提示词实验室 - 快速开始指南

## 🚀 5分钟快速上手

### 第一步：配置API（必须）

1. 进入"设置"页面
2. 找到AI配置部分
3. 填入您的API信息：
   ```
   API地址：https://api.deepseek.com/v1
   API密钥：sk-your-api-key-here
   ```
4. 保存设置

> 💡 提示：提示词实验室使用全局API地址和密钥，但每个角色可以配置不同的模型名称

### 第二步：配置AI角色（可选）

默认已有三个AI角色，都使用 `deepseek-chat` 模型：
- 🔍 分析师
- ⚠️ 批评家
- ✨ 优化师

如需添加更多角色或使用不同模型：
1. 点击"添加"按钮或编辑现有角色
2. 输入模型名称（如 `gpt-4`、`claude-3-5-sonnet`）
3. 选择角色类型
4. 保存

> 💡 提示：模型名称将作为API调用的model参数，可以为不同角色配置不同的模型

### 第三步：输入提示词

在"1. 输入提示词"框中输入您想优化的提示词，例如：
```
写一篇关于人工智能的文章
```

### 第四步：说明优化目标（可选）

在"2. 优化目标"框中说明问题，例如：
```
这个提示词太泛了，生成的内容没有重点
```

### 第五步：开始实验

1. 确保至少一个AI模型已启用（开关打开）
2. 设置讨论轮数（建议2轮）
3. 点击"开始实验"按钮

### 第六步：观察讨论

您会看到AI们依次发言：
- 🔍 **分析师**：分析提示词的优缺点
- ⚠️ **批评家**：指出问题和逻辑漏洞
- ✨ **优化师**：综合讨论，生成优化版本

### 第七步：查看结果

实验完成后，您会看到：
- ✅ 绿色卡片显示优化后的提示词
- 可以点击"应用此提示词"直接使用

### 第八步：继续优化（可选）

如果不满意：
1. 在反馈框中说明问题
2. 点击"进入下一轮优化"
3. AI团队会基于您的反馈继续改进

## 📝 示例场景

### 示例1：优化写作提示词

**原始提示词：**
```
写一篇文章
```

**优化目标：**
```
太简单了，没有任何指导
```

**AI讨论过程：**
- 分析师：指出缺少主题、风格、长度等关键信息
- 批评家：强调缺少目标受众和写作目的
- 优化师：生成详细的提示词

**优化结果：**
```
请撰写一篇1500字左右的科普文章，主题为"人工智能在医疗领域的应用"。
目标受众是对科技感兴趣的普通读者。
文章应包含：
1. 引人入胜的开头
2. 3-4个具体应用案例
3. 技术原理的通俗解释
4. 未来展望
语言风格：通俗易懂，避免过多专业术语
```

### 示例2：修复逻辑问题

**原始提示词：**
```
你是一个助手，帮我写代码，但不要写太复杂的，要写高质量的
```

**优化目标：**
```
"不要太复杂"和"高质量"有矛盾
```

**AI讨论过程：**
- 分析师：指出"复杂度"和"质量"的定义不清
- 批评家：强调这两个要求可能冲突
- 优化师：重新定义需求

**优化结果：**
```
你是一位经验丰富的编程助手。请帮我编写代码，要求：
1. 代码质量：遵循最佳实践，包含必要的注释和错误处理
2. 复杂度控制：优先使用标准库和常见模式，避免过度设计
3. 可读性：代码结构清晰，命名规范，易于理解和维护
在"简洁"和"完整"之间取得平衡，确保代码既高质量又易于理解。
```

## 🎯 最佳实践

### 1. 角色配置建议
- **至少3个角色**：分析师、批评家、优化师
- **不同角色类型**：确保有不同视角
- **不同模型**：可以为不同角色配置不同的模型，利用各模型的优势
- **API配置**：在设置中使用可靠的API服务

### 2. 提示词输入建议
- **清晰表达**：说明您想要什么
- **提供上下文**：说明使用场景
- **具体问题**：在优化目标中说明具体问题

### 3. 讨论轮数建议
- **1轮**：快速优化，适合简单问题
- **2轮**：标准配置，平衡效果和时间
- **3轮+**：复杂问题，需要深入讨论

### 4. 反馈技巧
- **具体说明**：不要只说"不好"，说明哪里不好
- **提供方向**：说明您期望的改进方向
- **逐步优化**：一次解决一个问题

## ⚙️ 高级功能

### 自定义AI角色

您可以添加更多AI角色并分配不同类型和模型：

1. **多个分析师**：使用不同模型从不同角度分析
   - 角色1：`gpt-4` - 分析师
   - 角色2：`claude-3-5-sonnet` - 分析师
2. **专业批评家**：专注特定领域
   - `deepseek-chat` - 批评家
3. **创意团队**：多个创造者头脑风暴
   - `gpt-4` - 创造者
4. **最终审核**：额外的优化师做最后把关
   - `claude-3-5-sonnet` - 优化师

> 注意：所有角色共用设置中的API地址和密钥，但可以使用不同的模型名称

### 历史记录管理

- **查看历史**：点击"历史记录"按钮
- **加载会话**：点击历史记录的"加载"按钮
- **继续优化**：加载后可以继续讨论
- **删除记录**：清理不需要的历史

### 批量优化

虽然界面一次只能优化一个提示词，但您可以：
1. 优化第一个提示词
2. 保存结果
3. 输入下一个提示词
4. 重复过程
5. 通过历史记录查看所有结果

## 🔧 故障排除

### 问题：AI没有响应
**解决方案：**
- 检查设置页面的API密钥是否正确配置
- 检查API地址是否正确
- 检查网络连接
- 查看浏览器控制台错误信息
- 确认API服务是否正常

### 问题：优化结果不理想
**解决方案：**
- 增加讨论轮数
- 添加更多AI模型
- 在优化目标中提供更详细的说明
- 使用反馈功能继续优化

### 问题：历史记录丢失
**解决方案：**
- 历史记录存储在浏览器localStorage中
- 不要清除浏览器数据
- 考虑定期导出重要的优化结果

### 问题：API费用过高
**解决方案：**
- 减少讨论轮数
- 减少启用的AI角色数量
- 在设置中切换到更经济的API服务
- 在优化目标中更明确，减少重复优化
