# 历史事件展示功能说明

## 功能概述

在小说列表的图谱弹窗中新增了"历史事件"Tab页，用于展示小说中所有抽取的关键事件。

## 修改内容

### 后端修改（Neo4jGraphService.java）

**位置**：`getGraphData` 方法（第2477-2557行）

**新增内容**：
- 查询所有Event节点
- 返回事件的详细信息：
  - `id`：事件ID
  - `summary`：事件摘要
  - `chapter`：所属章节
  - `importance`：重要性（0-1）
  - `emotionalTone`：情感基调
  - `tags`：标签数组
  - `description`：详细描述
  - `participants`：参与者列表
  - `location`：事件地点

### 前端修改（GraphDataModal.tsx）

**修改内容**：

1. **添加图标导入**：
   ```tsx
   import { HistoryOutlined } from '@ant-design/icons'
   ```

2. **更新GraphData接口**：
   ```tsx
   interface GraphData {
     events: any[]
     totalEvents: number
     // ... 其他字段
   }
   ```

3. **新增事件表格列定义**（第800-891行）：
   - 章节：显示事件所属章节
   - 事件摘要：事件的简短描述
   - 地点：事件发生地点
   - 参与者：参与事件的角色
   - 重要性：0-1的重要性评分（颜色标识）
   - 情感基调：positive/negative/neutral/tense
   - 标签：事件的分类标签

4. **新增历史事件Tab页**（第987-1020行）：
   - 显示事件总数
   - 支持展开查看详细描述
   - 支持按章节、重要性排序
   - 分页显示（每页10条）

### 表格列

|------|----------|------|--------|--------|----------|------|
| 第10章 | 主角突破到金丹期 | 玄天峰 | 林晨 | 0.90 | positive | 突破、修炼 |

### 重要性颜色标识
- **红色**：importance >= 0.8（非常重要）
- **橙色**：importance >= 0.6（重要）
- **默认**：importance < 0.6（一般）

### 情感基调颜色
- **绿色**：positive（积极）
- **红色**：negative（消极）
- **橙色**：tense（紧张）
- **默认**：neutral（中性）

### 可展开详情
点击行可展开查看事件的详细描述。

## 使用场景

### 1. 查看小说事件脉络
- 按章节顺序查看所有关键事件
- 了解剧情发展的主线

### 2. 检查事件质量
- 查看AI抽取的事件是否准确
- 检查重要性评分是否合理
- 确认地点和参与者信息是否正确

### 3. 分析事件分布
- 查看每章抽取了多少事件
- 分析事件的重要性分布
- 了解情感基调的变化

### 4. 辅助剧情规划
- 回顾已发生的关键事件
- 为后续章节提供参考
- 避免剧情重复或遗漏

## 注意事项

### 1. 事件数量控制
根据优化策略，每章最多抽取2-3个关键事件：
- importance >= 0.7
- 只记录长期影响的事件
- 不记录日常对话、普通战斗等

### 2. 历史数据兼容
- 旧数据可能缺少某些字段（如description、location）
- 前端会显示为"-"或"无"
- 不影响正常展示

### 3. 性能考虑
- 使用分页显示（每页10条）
- 支持按章节、重要性排序
- 大量数据时不会影响性能
