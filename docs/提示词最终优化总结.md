# 章纲生成提示词最终优化总结

## 优化原则

**核心理念**：设计巧妙而非堆积，通用而非具体

## 优化前后对比

### 优化前的问题
- ❌ 堆积具体剧情示例（如"离婚当天发现怀孕"）
- ❌ 冗长的描述和重复说明
- ❌ 缺乏通用性，限制了AI的创造空间

### 优化后的特点
- ✅ 精简为核心原则，删除具体剧情示例
- ✅ 保持通用性，适用于各种题材
- ✅ 结构清晰，层次分明

## 最终提示词结构

### 1. 角色与任务（4条核心目标）
- 转折设计
- 节奏控制
- 钩子布局
- 核心看点

### 2. 卷首三章特别要求（精简为4条）
- 第1章：高信息量强冲突开局
- 第1-2章：外显反击
- 第1-3章：场景展示能力
- 第1-3章：钩子分布

### 3. 章纲质量核心原则（5条）
1. **转折设计**：意料之外、情理之中
2. **冲突分层**：情绪<利益<生死
3. **能力展示**：场景行为，禁止旁白
4. **外显原则**：可观察行为，禁止隐蔽
5. **Show, Don't Tell**：禁止内心独白

### 4. 标签规范（精简版）
**必须包含**：
- [事件]
- [冲突]：对立双方+争夺焦点+实质赌注
- [结果]
- [看点]：具体可视化
- [收束]：外显行为，严禁内心独白

**可选标签**：
- [承接]
- [推进]
- [钩子]

### 5. 结尾风格（精简为3类+1条建议）
- SOFT_CLOSE
- EMOTIONAL_BEAT
- STRONG_HOOK
- 使用建议（1行）

## 关键优化点

### 删除的内容
1. ❌ 所有具体剧情示例
2. ❌ 冗长的"禁止-改进"对照
3. ❌ 重复的说明和解释

### 保留的精华
1. ✅ 核心原则（5条）
2. ✅ 标签规范（清晰明确）
3. ✅ Show, Don't Tell（重点强调）

## 提示词字数对比

- **优化前**：约2000字
- **优化后**：约1200字
- **精简率**：40%

## 核心突破

### 最重要的一条：Show, Don't Tell

**禁止**：
- "她想..."
- "她心中..."
- "她感到..."
- "她决定..."

**要求**：
- 只写可观察的外部行为
- 让读者自己推断
- 通过矛盾行为制造悬念

### 实战效果

**优化前生成的章纲**：
```
[收束] 姜月初在内心对自己说："沈修谨，我们结束了。"
```

**优化后应生成**：
```
[收束] 姜月初没有回应，只是默默删除了他的联系方式，然后拨通了律师的电话。
```

## 通用性设计

### 适用题材
- ✅ 现代言情
- ✅ 古代言情
- ✅ 玄幻修仙
- ✅ 都市重生
- ✅ 悬疑推理
- ✅ 科幻未来

### 设计巧妙之处

1. **冲突分层**：情绪<利益<生死
   - 不限定具体冲突类型
   - 给出优先级指导
   - 适用所有题材

2. **能力展示**：场景行为
   - 不限定具体能力
   - 强调展示方式
   - 通用于所有金手指

3. **Show, Don't Tell**
   - 不限定具体情节
   - 强调表现手法
   - 适用所有场景

## 最终效果预期

使用优化后的提示词，生成的章纲应该：

1. **无内心独白**：所有情绪通过行为暗示
2. **有意外转折**：关键节点不完全线性
3. **冲突有赌注**：不只是情绪冲突
4. **看点可视化**：具体行为而非抽象描述
5. **保持悬念**：角色想法"藏"起来

## 检查清单

生成章纲后快速检查：
- [ ] 是否有"她想/她心中/她感到"？→ 必须删除
- [ ] [冲突]是否有实质赌注？→ 不能只是情绪冲突
- [ ] [看点]是否具体可视化？→ 不能是抽象描述
- [ ] [收束]是否用外显行为？→ 不能是内心独白
- [ ] 是否有意外转折？→ 不能完全线性
